CAN Bus Monitoring & BMS Data Emulation

This academic project focuses on the design and implementation of a CAN-based embedded communication system for real-time monitoring and emulation of Battery Management System (BMS) data. The project was developed in collaboration with Amani Salah.

The system architecture is composed of two STM32 boards connected via a CAN bus. The STM32F407 acts as a CAN gateway, while the STM32F746 emulates a BMS node responding to CAN request frames. Communication is handled using extended CAN identifiers and frame filtering.

A Qt-based graphical interface running on the PC sends CAN request frames for parameters such as: State of Charge (SOC), total voltage and current, maximum/minimum cell voltages, and temperature values. Each response frame is decoded, mapped by CAN ID, and visualized in real time.

An Arduino board is used solely as a UART bridge between the embedded system and the Qt interface. This project significantly strengthened my understanding of CAN protocol internals, real-time embedded communication, and hardware–software co-design.
